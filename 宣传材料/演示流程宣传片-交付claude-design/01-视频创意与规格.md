# 01 · 视频创意与规格

## 一句话
做一个**横屏网页宣传片**：一侧是**竖屏手机**，里面**动态演示"智评慢病"小程序从主页→问答→报告→自由对话→进阶评估的完整使用流程**；另一侧是**解说面板**，用文字讲清"它是什么 / 怎么用"。整页自动播放，**它的播放过程本身就是视频**。

## 这是什么产品（一句话背景）
"智评慢病"是一个 **chat-first 的慢病风险自测小程序**：用户像聊天一样回答问题，得到 **糖尿病 / 高血压 / 心血管病 未来 5 年**的风险评估（基于房山家系队列 Cox 模型，计算在本机完成），并能继续自由提问获取生活方式建议。

## 核心创意
- **左右分屏**：横屏 1920×1080。一侧"竖屏手机"跑小程序演示，另一侧解说面板（标题 + 要点）。手机与面板**同步**——手机演到哪个能力，面板就点亮对应那句话。
- **演示是"套壳"**：手机里的对话、反馈、计算、报告**不是截图、不是跑真实接口**，而是用 HTML/CSS **重新画出真实界面**并按脚本**演出来**（打字、点选、确认条弹出、数字滚动、报告展开……）。所有界面与文案的还原标准见 `02-小程序UI还原规范.md`，分步脚本见 `content/demo-script.json`。
- **数据真实**：报告里的百分比、Top5、模型透明度等数字来自冻结医学内核的真实计算（`content/report-data.json`），**请勿手改**。

## 风格
- **科技风 + 医疗高级感**：蓝青冷色（`#4F8CFF / #22D3EE / #7DD3FC`）、玻璃拟态、冷色光晕阴影、细网格/光流背景、顺滑缓动（`cubic-bezier(.22,1,.36,1)`）、按钮流光。
- **表现力强但克制**：转场用位移/缩放/流光/计数动画即可，**不要满屏粒子轰炸**。
- **降低 AI 味**：**不要**"赋能 / 智能化 / 开启未来 / 一键 XX"这类空话套话。文案已按产品真实语气写好（见 `narration-panels.json` 与脚本），**直接使用**。
- **不做字幕**：所有要讲的话都以**解说面板文字**呈现（"配音内容上屏"）。本片默认**不配音**；若日后要配，再单独处理。

## 规格
- 尺寸：**1920×1080（16:9）**，横屏。
- 时长：**约 45–50 秒**（脚本编排为 ≈ 48s，见 `03-分镜脚本与时间轴.md`）。
- 字体：**Noto Sans SC**（务必内嵌/打包，离线可渲染中文）。
- 交付物：**一个自包含、可离线打开的 `promo.html`**（所有图片/字体内联或随附 `assets/`，双击即播放）。

## ★ 技术要求：必须"可逐帧确定性捕获"（用于转成视频）
做完后我会用无头浏览器把这个网页**逐帧截图**再合成 MP4。为保证清晰、不抖动，请遵守：
1. **单一主时钟**：整页动画由一条主时间轴 `t`（毫秒/秒）驱动，渲染应尽量是"**t 的纯函数**"——给定 `t` 就能确定该帧画面。避免依赖真实墙钟的随机动画。
2. **自动播放**：页面加载即从 `t=0` 播放到结束（约 48s）。
3. **用 requestAnimationFrame 推进 `t`**（标准写法即可）；不要用无法被接管的独立计时器画关键动画。环境装饰性动效（背景光斑、打字点等）即使用 CSS 动画也没关系，但**关键剧情**请走主时钟。
4. **固定 1920×1080 舞台**：用一个 1920×1080 的根容器，靠 `transform: scale()` 适配窗口（letterbox），不要依赖窗口实际尺寸决定布局。
5. 建议提供键盘控制：`空格`=播放/暂停、`R`=重播、`F`=全屏；并可隐藏任何控制条（截图时不出现）。
6. 给挂载完成一个稳定标志（如某个根节点 `data-ready="1"`），方便我判断"可以开始截帧了"。

> 满足以上几点，我就能把它干净地转成 30fps 的 MP4。无需你处理视频导出。
