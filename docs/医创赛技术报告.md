# 健康风险助手 · AI Health Copilot
## 医学创新大赛技术报告

> **一句话定位**：面向普通居民的「对话式」慢病风险评估与生活方式科普 AI 助手——像聊天一样几分钟读懂未来 5 年的 **2 型糖尿病 / 高血压 / 心血管病** 风险，并获得**可解释、可溯源、高度保守**的生活方式建议。

> **第一原则**：所有医学风险计算 100% 在本地完成、与原始统计模型逐位一致；AI 只负责引导、解释、科普与生活方式建议，**绝不参与风险计算，绝不输出药物 / 剂量 / 诊断 / 治疗方案**。

---

## 摘要

我国慢性病防控正面临"患病率高、知晓率/控制率低、基层筛查与健康教育资源不足"的结构性矛盾。传统的"问卷表单 → 一次性出分"风险工具，普遍存在**交互枯燥、可解释性差、科普不可信、缺乏安全边界**等问题，难以转化为居民真正的行为改变。

本项目将一套基于**房山家系队列（2016—2024）Cox 比例风险模型**的慢病风险预测能力，重构为**chat-first 的 AI 健康风险助手**：用对话式渐进问诊替代大表单，用可视化的 Top-5 贡献度与逐层 C-index 提供可解释性，用带来源与年份的循证知识库（RAG）提供可溯源的科普问答，并以"冻结医学内核 + 三道安全护栏 + 隐私本地计算"确保医学严谨性与合规性。

核心技术贡献有四：① **医学计算与 AI 彻底解耦**——风险内核被 golden 测试锁定、字节不可改，AI 永不接触可重算风险的原始指标；② **chat-first 渐进式问诊编排**——确定性调度 + 实时定性解读，移动端体验顺滑且永远可用（AI 不可用即降级）；③ **居民安全版 RAG 知识体系**——构建期即剔除药物/诊断/处方，238 个知识块全部带溯源字段；④ **高度保守的多层安全边界**——系统 Prompt 护栏 + 域护栏 + 出口正则过滤，从源头杜绝越界。

## 1. 立项背景与临床痛点

### 1.1 慢病防控的现实压力

2 型糖尿病、高血压、心血管病是我国疾病负担最重的慢性病。其防控的关键在于**早期风险识别 + 长期生活方式干预**，而非单纯依赖药物。然而现实中存在三道鸿沟：

| 鸿沟 | 表现 | 后果 |
|---|---|---|
| **筛查鸿沟** | 高危人群在出现症状前缺乏便捷的风险自评手段 | 错过可逆窗口期，确诊即并发症 |
| **认知鸿沟** | 居民"知道要健康"，但不知道**自己**风险高在哪、该先改什么 | 健康建议泛化、无法落地 |
| **信任鸿沟** | 网络健康信息良莠不齐，AI 工具易"一本正经地编造"甚至给出危险的用药/诊断建议 | 误导风险，难以进入严肃健康场景 |

### 1.2 现有同类产品的不足

- **表单式风险工具**：交互枯燥、填写门槛高、移动端体验差；结果是一个孤立的分数，缺乏"为什么"与"接下来怎么办"。
- **通用健康问答 AI**：缺乏个体风险背景，建议泛化；更严重的是**安全边界缺失**——可能输出药物名称、剂量、诊断断言，存在医疗与合规风险。
- **黑箱风险评分**：不公开模型口径与因素贡献，可解释性与可信度不足，难以被医生与居民同时接受。

### 1.3 本项目的回应

> 把"枯燥表单 + 黑箱分数 + 不可信科普"升级为 **"对话问诊 → 可解释风险画像 → 可溯源生活方式建议"**，并以工程手段确保**医学计算不被 AI 污染、回答不越医学伦理边界、隐私不离开设备**。

## 2. 核心创新点（创新性）

| # | 创新点 | 解决的问题 | 关键做法 |
|---|---|---|---|
| **C1** | **chat-first 渐进式问诊** | 表单枯燥、门槛高、移动端体验差 | 像聊天一样逐题作答，三层渐进（基础→血液→影像），随时可跳过/直接自由聊天；不点输入框不弹键盘 |
| **C2** | **医学计算与 AI 解耦（冻结内核）** | AI "幻觉"污染风险数值、模型不可复现 | 风险内核 4 文件加 `FROZEN` banner、统一入口、golden 快照测试逐位锁定；**AI 永不参与计算** |
| **C3** | **可解释 + 可溯源** | 黑箱评分、科普不可信 | Top-5 因素贡献度可视化 + 逐层 C-index + Cox 公式透明；科普回答带**来源机构与年份** |
| **C4** | **高度保守的安全边界** | AI 越界给药物/诊断/治疗 | 系统 Prompt 护栏 + 域护栏 + 出口正则过滤三道防线；**知识库构建期即剔除**药物/剂量/诊断/处方 |
| **C5** | **隐私本地计算 + 可离线降级** | 健康数据外泄、强依赖后端 | 风险计算 100% 在浏览器本地，只上传"风险等级 + Top5 标签"；后端不可用时自动降级为确定性流程，应用始终可用 |

> **最具差异化的一点（C2）**：业内常见做法是"让大模型直接给风险/建议"。本项目反其道而行——**风险由经统计学验证的 Cox 模型本地计算并被测试冻结，AI 只在其外层做引导与解释**。这既保证了医学严谨性与可复现性，又规避了大模型在高风险医疗场景的幻觉问题。

## 3. 临床方法学与预测模型（方法学深度）

### 3.1 数据来源与建模流程

- **队列**：房山家系队列，随访区间 **2016—2024**，以家系为单位入组。
- **建模管线**：
  1. **质量控制（QC）**：变量清洗、异常值处理、单位统一（系数表中变量以 `_qc` 标识为质控后变量）。
  2. **变量筛选**：弹性网络（Elastic Net，α=0.5）在 L1/L2 之间平衡，兼顾稀疏性与共线性稳健。
  3. **回归建模**：Cox 比例风险回归，采用**稳健标准误并按家系聚类校正**（家系内相关性）。
  4. **嵌套验证**：构建**三层嵌套模型**（基础信息 / +血液检查 / +血管影像），逐层评估判别力（C-index）与重分类改善（NRI / IDI）。

### 3.2 预测公式

个体在时间 $t$ 的发病风险：

$$\text{Risk}(t) = 1 - S_0(t)^{\exp(LP)}, \qquad LP = \sum_i \beta_i\,(x_i - \bar{x}_i)$$

- $S_0(t)$：基线生存函数（人群均值水平下的生存概率）；
- $\beta_i$：Cox 系数，等于 $\ln(HR)$；$x_i$ 为个体取值，$\bar{x}_i$ 为队列人群均值；
- **缺失处理**：未填写的指标取**人群均值** $\bar{x}_i$，中心化后该项贡献 $\beta_i(x_i-\bar x_i)=0$——即"不填不偏倚"，使分层渐进式问诊在数学上自洽。

### 3.3 三个预测结局（未来 5 年新发风险）

| 结局 | 基线生存率 $S_0(5y)$ | 样本量 | 事件数 | C-index（L1 / L2 / L3） |
|---|---|---|---|---|
| 新发 2 型糖尿病 | 0.94768 | 3,904 | 961 | 0.667 / 0.722 / 0.733 |
| 新发心血管病 | 0.78173 | 3,396 | 1,898 | 0.626 / 0.687 / 0.687 |
| 新发高血压 | 0.79549 | 2,936 | 818 | 0.647 / 0.728 / 0.750 |

> C-index 随指标层级递增而提升，定量印证了"三层渐进式问诊——补充越多、评估越准"的设计合理性。系统同时保留一个 **10 年全因死亡**辅助模型（$S_0(10y)=0.93469$）用于内部对照，前端不直接展示。

### 3.4 代表性变量与风险比（HR = exp(β)）

下表节选**应用实际采集**的关键变量，HR 为多变量校正后的偏效应（连续变量为每 +1 单位、二分类变量为"有 vs 无"）：

**新发 2 型糖尿病**

| 变量 | β | HR | 解读 |
|---|---|---|---|
| 糖化血红蛋白 HbA1c | 0.1193 | 1.127 | 每升高 1%，风险 ↑约 12.7% |
| 空腹血糖 FBG | 0.0752 | 1.078 | 糖代谢异常的核心前兆 |
| LDL-C（坏胆固醇） | 0.6299 | 1.877 | 校正后强相关 |
| 糖尿病家族史 | 0.3689 | 1.446 | 遗传易感（不可改变） |
| 腰围 | 0.0110 | 1.011 | 腹型肥胖，每 +1cm |
| 运动量(MET·h/周) | −0.0032 | 0.997 | 保护因素 |
| HDL-C（好胆固醇） | −0.2145 | 0.807 | 保护因素 |

**新发心血管病**

| 变量 | β | HR | 解读 |
|---|---|---|---|
| HDL-C（好胆固醇） | −0.6336 | 0.531 | 强保护因素 |
| LDL-C | 0.2766 | 1.319 | 动脉粥样硬化驱动 |
| CCA-IMT（颈动脉内中膜厚度） | 0.2230 | 1.250 | 亚临床动脉硬化标志 |
| 脑卒中家族史 | 0.1117 | 1.118 | 遗传背景 |
| 年龄 | 0.0151 | 1.015 | 每 +1 岁 |

**新发高血压**

| 变量 | β | HR | 解读 |
|---|---|---|---|
| 性别（男=1） | 0.3504 | 1.420 | 男性风险更高 |
| 收缩压 SBP | 0.0142 | 1.014 | 每 +1 mmHg |
| 既往心血管病 | 0.5162 | 1.676 | 共病关联 |
| HbA1c | 0.0857 | 1.089 | 代谢联动 |
| HDL-C | −0.4655 | 0.628 | 保护因素 |

> **方法学诚实声明**：个别系数方向与单因素直觉相反（如校正其他血脂后 TC 的偏效应为负、吸烟对高血压的校正偏效应为负），这是**多变量模型在相关预测因子并存时的正常偏效应表现**，不应作因果解读。模型以**整体判别力（C-index）与校准**为评估口径，单一系数仅用于贡献度排序与解释展示。

### 3.5 完整模型特征 vs 应用采集子集

完整 tier-3 系数集包含 40+ 特征（含教育、职业、收入、身高体重分项、心率、ApoB、hsCRP、ALT、Hcy、体水分、缺失指示变量等）。**应用前端只采集其中临床意义清晰、居民可自报或体检常见的子集**（约 30 项，分三层）；其余特征对每位用户按人群均值中心化（贡献 0）。该设计在"问诊负担"与"预测精度"之间取得平衡，且与公式 3.2 的缺失处理天然一致。

**三层渐进式指标设计**

| 层级 | 指标 | 临床意义 | 获取门槛 |
|---|---|---|---|
| **① 基础信息** | 年龄、性别、BMI（身高体重换算）、腰围、睡眠、静坐、运动量、吸烟、饮酒、家族史、既往病史 | 可改变生活方式 + 遗传背景 | 居民自报，零门槛 |
| **② 血液检查** | 血压、血糖(FBG/HbA1c)、血脂(TC/HDL/LDL/TG/ApoA)、用药 | 代谢与靶器官状态 | 常规体检报告 |
| **③ 血管影像** | ABI、baPWV、CCA-IMT | 亚临床动脉硬化 | 专项检查（做过则填） |

### 3.6 诊断阈值排除与风险分级

- **诊断阈值排除**：当指标已达临床诊断标准，系统**不再对该结局做"新发风险"预测**，而是提示就医确认——
  - 高血压：SBP ≥ 140 或 DBP ≥ 90 或正在服用降压药；
  - 糖尿病：FBG ≥ 7.0 mmol/L 或 HbA1c ≥ 6.5% 或正在服用降糖药。
- **风险分级**：**高危 > 20%、中危 > 10%、低危 ≤ 10%**（5 年新发概率）。

### 3.7 模型局限（如实陈述）

- 单一队列建模，**尚需多中心、外部人群前瞻验证**；
- 心血管结局 C-index 处于中等区间（0.63–0.69），用于**人群分层与健康教育**适宜，**不作个体确诊依据**；
- 系数为特定队列的关联估计，**外推到其他地区/年龄结构需谨慎**。

## 4. 系统设计与技术实现（技术架构）

### 4.1 总体架构

四层结构，AI 只读不写"医学内核"：

```
┌──────────────────────────────────────────────────────────────┐
│  UI 层（React）   高级感首页 · 对话线程 · 健康风险报告           │
├──────────────────────────────────────────────────────────────┤
│  AI 外壳          对话编排 · 流式渲染 · 前端护栏兜底（可降级）    │
├──────────────────────────────────────────────────────────────┤
│  编排层           确定性问诊调度 + store(reducer) + 本地 Cox 计算 │
├──────────────────────────────────────────────────────────────┤
│  医学内核(冻结)    riskConfig / riskEngine / validation …  字节不改│
└──────────────────────────────────────────────────────────────┘
                   │ 仅传：风险等级 + Top5 因子标签
                   ▼
┌──────────────────────────────────────────────────────────────┐
│  后端代理(serverless / Node)  /api/extract  /api/answer(SSE)    │
│  ① 域护栏 ② BM25 检索 ③ 组装 Prompt ④ DeepSeek 生成 ⑤ 出口过滤  │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 chat-first 对话编排

- **单一状态源**：`store.jsx`（`useReducer + Context`）。`messages` 是对话线程的唯一显示来源；`pending`（"Composer 描述符"）驱动底部输入区该渲染什么（数字 / 选项 / BMI / 运动 / 分支 / 自由问答）。
- **确定性问诊调度**：`questionFlow.js` 从内核的 `VARIABLES` 派生逐题顺序与口语化提示，决定"下一题"；本层答完即给**定性解读（绝不含风险数值）**，满足"不提前剧透风险值"。
- **闭环**：reducer 设 `pending` → Composer 渲染对应输入 → 用户操作 dispatch → reducer 追加消息并算出下一个 `pending`。

### 4.3 人性化交互细节

- **智能理解 + 答题确认**：用户口语化输入（如"腰围 50 到 60"）由 AI 抽取为数值，并以确认条回显"已记录 · 腰围：55cm"，让用户清楚**自己提供的信息如何被纳入模型**。
- **身高体重自动算 BMI**：不直接问 BMI，而是问身高、体重，由助手实时换算并告知 BMI 与中文分类（偏瘦/正常/超重/肥胖）。运动量同理（高/低强度时长 → MET·h/周）。
- **两种进入模式**：① 正式问诊（答题→报告→自由提问）；② **直接自由聊天**（首页与问诊页均提供入口），随时可一键切换回正式评估。
- **开场即声明身份与边界**："我是健康生活方式助手，不提供诊断或治疗方案，内容仅供参考，不替代医生。"

### 4.4 健康风险报告

一句话总结 + 三疾病卡片（风险值/等级/简释）+「为什么会这样」+ **Top-5 贡献度可视化**（Apple Health 风）+ **模型透明度**（逐层 C-index / Cox 公式）+ 诊断提示 + 免责声明。

### 4.5 AI 层与 RAG 知识体系

- **模型**：DeepSeek（OpenAI 兼容，`deepseek-chat`），仅用于文本理解、科普、解释；密钥只存服务端环境变量。
- **两个端点**：`/api/extract`（自由文本 → 指标值，JSON 模式）、`/api/answer`（分层解读 /「为什么」/ 自由问答，**SSE 流式**）。
- **检索（BM25-lite，零依赖、可离线）**：针对中文采用 ASCII 词 + CJK **一元/二元 gram** 分词；BM25（k1=1.5, b=0.75）；查询 = 用户问题 + 疾病 + Top5 因子标签 → 按 `disease_scope` 过滤 → 取 Top-5 注入提示词 →回答末尾追加**来源与年份**。
- **知识库构建**（`rag_knowledge_base/` → 产出 `api/rag/chunks.json`）：抓取（指南 PDF / WHO 实况报道）→ PyMuPDF 转 Markdown（保留页码）→ **安全过滤 + 分块**（整节/段落剔除药物/剂量/诊断/手术/处方，按标题分知识点、过长软切 ~700 字）→ 带完整溯源字段的 chunk。

| 知识库统计 | 数值 |
|---|---|
| 文档数 / 分块数 | 17 篇 / **238 块** |
| 按类型 | 权威指南 165 · 公共卫生科普 54 · 内部文档 12 · 文献综述 7 |
| 按疾病（块计） | 心血管 170 · 糖尿病 138 · 高血压 129 |
| 核验结论 | **零药物/剂量/手术/处方残留**，每块均带 `source_url` + `year` |

| 维度 | 做法 | 居民可见 |

## 6. 安全合规与医学伦理

### 6.1 三道安全护栏

1. **系统 Prompt 即护栏**：只做生活方式建议、健康教育、指标与风险解释、就医提醒；**禁止**药物 / 剂量 / 诊断 / 治疗方案 / 停换药 / 处方 / 急诊指导；只依据【参考资料】回答，资料不足时给保守通用建议并建议咨询专业人员。
2. **域护栏**：无关问题（时政、人物、编程、闲聊）统一回绝，不展开。
3. **出口安全过滤**：对模型累计输出做正则扫描（药名 / 剂量 `mg·每日X次` / 诊断断言"确诊为 / 你患有"），命中即替换为安全话术并结束。

> **双重剔除**：知识库**构建期**即移除临床用药/诊断/处方内容，**出口期**再扫描兜底——两道闸门确保"即使模型想说，也无可越界的素材，且说了也会被拦"。

### 6.2 隐私与数据合规

- **风险计算永远在浏览器本地**完成；不收集姓名、身份证号等身份信息。
- 离开设备的只有"聊天文本 + 风险等级 + Top5 因子标签"，**不含可反推/重算风险的原始指标**。
- 后端密钥（`DEEPSEEK_API_KEY`）只存服务端环境变量，**仓库与前端打包产物零泄漏**。

### 6.3 伦理定位与免责

- 明确定位为**健康教育 / 风险参考工具**，**非医疗器械、非诊断系统**；每次评估与回答均附"以上为生活方式参考，不替代专业诊疗"。
- 命中诊断阈值即**主动提示就医**，而非继续给出"风险分数"造成误导。
- 对模型外推局限、个体差异保持透明，倡导"风险评估 → 与医生沟通 → 规范随访"的健康路径。

## 7. 质量保障与工程可靠性

- **Golden 基线测试**：`cox.golden.test.js` 对"全均值 / 低危 / 高危 / 触发诊断"四组 fixture 快照风险输出（值/等级/线性预测/Top 贡献/建议），**逐位证明医学计算 100% 不变**；严禁盲目更新快照。
- **测试覆盖**：内核 golden + 问诊调度 + BM25 检索，共 **21 用例**全部通过。
- **CI/CD**：推送 `main` 触发 GitHub Actions 构建并部署到 Pages。
- **永远可用**：每条 AI 路径都有确定性回退（抽取失败→追问；解释失败→本地 `buildWhy`），**后端不可用时应用仍可完整完成评估**。

## 8. 应用价值与推广前景

### 8.1 应用场景

| 场景 | 价值 |
|---|---|
| **基层公共卫生 / 家庭医生签约** | 居民自评 + 高危分层，辅助随访优先级排序，减轻基层问卷负担 |
| **体检中心** | 体检报告"一键解读"，把孤立指标转化为可理解的风险画像与行动建议 |
| **企业 / 社区健康管理** | 员工/居民慢病风险普查与健康科普，低成本规模化触达 |
| **慢病健康教育** | 可溯源、保守、个性化的生活方式科普，提升健康素养 |

### 8.2 社会效益

- 把"被动就医"前移为"主动风险认知 + 生活方式干预"，契合**关口前移、预防为主**的慢病防控国策；
- 以**可解释、可溯源、安全保守**的设计，缓解 AI 健康工具的信任与合规顾虑。

### 8.3 落地与可持续性

- **轻量可嵌入**：单页 Web App，可作为 H5 / 小程序 WebView 接入既有健康平台；前端纯静态，国内 Pages 可访问。
- **零依赖可私有化**：后端为零外部依赖的 Node 服务，可部署到 Render / 腾讯云 SCF / 阿里云 FC，便于**院内私有化与数据本地化**。
- **可持续迭代**：知识库新增文献后重跑构建脚本即可扩充；模型系数表与前端解耦，更新内核仅需同步 golden 基线。

## 10. 结论

本项目以"**医学计算冻结 + AI 解释外壳**"为核心范式，证明了在严肃健康场景中，大模型可以在**不触碰风险计算、不越医学伦理边界**的前提下，显著提升慢病风险评估的**交互体验、可解释性与科普可信度**。系统在方法学上以经统计学验证的 Cox 模型为基石、以 golden 测试锁定可复现性，在工程上以三道护栏与本地计算保障安全合规，在产品上以 chat-first 与可溯源科普实现"读懂风险 → 知道为什么 → 知道怎么做"的健康教育闭环，具备清晰的临床价值与可落地的推广路径。
