# AI Health Copilot · 项目技术报告

> 慢性病风险评估 + 生活方式科普的 **chat-first AI 健康风险助手**。
> 在不改动任何医学计算的前提下，把原"表单 → 结果"工具升级为"对话 → 风险画像 → 可溯源的生活方式建议"。

---

## 1. 项目概述

| 维度 | 说明 |
|---|---|
| 定位 | 面向普通居民的慢病（2 型糖尿病 / 高血压 / 心血管病）风险评估与生活方式科普助手 |
| 形态 | 移动端优先的单页 Web App（chat-first），可作为 H5 / 小程序 WebView 接入 |
| 模型 | 风险预测：房山家系队列 **Cox 比例风险模型**（本地计算）；AI 解释/问答：**DeepSeek**（`deepseek-chat`）+ **RAG 知识库** |
| 第一原则 | **保留全部医学计算逻辑，模型输出与原版 100% 一致**；AI 只做引导/解释/科普/生活方式建议，**绝不参与风险计算** |
| 在线地址 | GitHub Pages（国内可访问，接国内后端）/ Vercel（完整版） |

设计目标：

1. **Chat-first**：像聊天一样逐题问诊，实时定性解读，最后给风险报告与建议，而非一次性大表单。
2. **可解释 + 可溯源**：每个风险因素、每条建议都能追到来源（C-index、Top-5 贡献度、知识库文献年份）。
3. **高度保守的安全边界**：只谈生活方式与风险解释，**绝不输出药物 / 剂量 / 诊断 / 治疗方案**。

## 3. 医学计算内核（冻结层）

### 3.1 预测模型

- **队列**：房山家系队列，随访 2016—2024。
- **方法**：Cox 比例风险回归（稳健标准误、按家系聚类校正）+ 弹性网络变量筛选（α=0.5）+ 三层嵌套模型逐层 C-index / NRI / IDI 评估。
- **预测公式**：

$$\text{Risk}(t) = 1 - S_0(t)^{\exp(LP)}, \quad LP=\sum_i \beta_i (x_i - \bar{x}_i)$$

其中 $S_0(t)$ 基线生存函数、$\beta_i$ 为 Cox 系数 $\ln(HR)$、$x_i$ 个体值、$\bar{x}_i$ 人群均值。缺失指标取人群均值（中心化后贡献为 0）。

### 3.2 三个结局（未来 5 年新发风险）

| 结局 | 基线生存率 S₀(5y) | 样本量 | 事件数 | C-index（L1 / L2 / L3）|
|---|---|---|---|---|
| 新发糖尿病 | 0.94768 | 3904 | 961 | 0.667 / 0.722 / 0.733 |
| 新发心血管病 | 0.78173 | 3396 | 1898 | 0.626 / 0.687 / 0.687 |
| 新发高血压 | 0.79549 | 2936 | 818 | 0.647 / 0.728 / 0.750 |

### 3.3 三层渐进式指标

- **第一层 基础信息**：年龄、性别、BMI、腰围、睡眠、静坐、运动量(MET·h/周)、吸烟、饮酒、家族史、既往病史。
- **第二层 血液检查**：血压(SBP/DBP)、血糖(FBG/HbA1c)、血脂(TC/HDL/LDL/TG/ApoA)、用药情况。
- **第三层 血管影像**：ABI、baPWV、CCA-IMT。

### 3.4 诊断阈值与风险分级

- 诊断提示（达到即提示就医、跳过新发预测）：高血压 SBP≥140 / DBP≥90 / 服降压药；糖尿病 FBG≥7.0 / HbA1c≥6.5 / 服降糖药。
- 风险分级：**高危 > 20%、中危 > 10%、低危 ≤ 10%**。

### 3.5 冻结保障

- 内核统一入口 `src/kernel/index.js`，4 个文件加 FROZEN banner。
- **Golden 测试** `src/kernel/__tests__/cox.golden.test.js`：对"全均值 / 低危 / 高危 / 触发诊断"四组 fixture 快照 `calcCoxRisk` / `riskLevel` / `getExclusionReason` / `checkDiagnostics` / `getHealthAdvice` 的输出。重构全程保持通过，逐位证明"输出 100% 不变"。

## 4. Chat-first 交互与编排

- **确定性问诊调度器**（`intake/questionFlow.js`）：从内核的 `VARIABLES` 派生逐题顺序与口语化提示，决定"下一题"，并做答案映射与校验。AI 不可用时自动降级为快捷选项/数字输入，应用始终可用。
- **store**（`app/store.jsx`，`useReducer + Context`）：`messages` 为显示单一来源，`pending` 驱动底部输入区。支持流式 AI 消息（`ai_stream`）。
- **分层定性解读**：每完成一层，本地算出 Top 贡献因子并给出**定性**解读（不出现任何风险数值），满足"不要提前输出风险值"。
- **Health Report**：一句话总结 + 三疾病卡片（风险值/等级/简释）+「为什么会这样」+ Top-5 贡献度可视化（Apple Health 风）+ 模型透明度（C-index/公式）+ 诊断提示 + 免责声明。

## 5. AI 层与安全边界（高度保守）

### 5.1 模型与调用

- **DeepSeek**（OpenAI 兼容），`deepseek-chat`，base `https://api.deepseek.com`。密钥只存后端环境变量。
- 两个端点：`/api/extract`（自由文本→指标值，JSON 模式）、`/api/answer`（分层解读/「为什么」/自由问答，**SSE 流式**）。

### 5.2 三道护栏

1. **系统 Prompt 即护栏**：只做生活方式建议、健康教育、指标与风险解释、就医提醒；**禁止**药物/剂量/诊断/治疗方案/停换药/处方/急诊指导；只依据【参考资料】回答；风险数值由系统给出，不自行计算。
2. **域护栏**：无关问题（时政、人物、编程、闲聊）统一回绝——"当前对话聚焦于健康风险评估与生活方式管理…"。
3. **出口安全过滤**：对模型累计输出做正则扫描（药名 / 剂量 `mg·每日X次` / 诊断断言"确诊为/你患有"），命中即替换为安全话术并结束。

> 风险数值不进 AI——只传"风险等级 + Top5 因子标签"，从源头杜绝 AI 改动或反推风险。

## 6. RAG 知识体系构建（重点）

知识库独立构建于 `rag_knowledge_base/`，产出的分块随后端部署在 `api/rag/chunks.json`。

### 6.1 来源范围与筛选规则

**纳入**（面向普通居民、生活方式与指标解释）：
- 权威指南：中国高血压/糖尿病防治、食养指南、居民膳食指南、身体活动指南。
- 公共卫生科普：WHO 中文实况报道。
- 文献：里程碑 RCT 与系统综述的结论性证据。
- 小程序内部文档：变量说明、C-index / Top-5 贡献度等透明度说明。

**禁止**（构建期即剔除）：药物、剂量、处方、临床诊断流程、手术、急诊/住院处理。

### 6.2 抓取与处理流程

```
抓取  PDF(curl) · HTML(curl+正则) · WHO 页面(转 Markdown)  → raw/ 留存原件
  │
处理  PDF→Markdown (PyMuPDF，保留页码；扫描版预留 OCR 接口，本批未触发)
  │   每文档加 YAML metadata
  ▼
安全过滤 + 分块  scripts/build_chunks.py
  ├─ 整节剔除：标题含 治疗/用药/诊断和治疗/手术/处方
  ├─ 段落剔除：药名 / 剂量 / 中医食药方剂量(用法用量·限执业医师使用)
  ├─ 噪声过滤：目录点引线 / 页码 / 无实义短块
  └─ 分块：以标题(H1/H2/H3)为知识点边界，过长按 ~700 字软切
  ▼
产出  processed/consumer_safe(居民安全版) · chunks/chunks.jsonl · metadata · excluded(排除清单)
```

### 6.3 分块与 metadata schema

每个 chunk 携带完整溯源字段：

```json
{
  "chunk_id": "chunk_0001",
  "doc_id": "who_zh_hypertension",
  "title": "高血压（实况报道）",
  "heading_path": "高血压 > 风险因素",
  "text": "……",
  "source_url": "https://www.who.int/zh/…/hypertension",
  "organization_or_journal": "世界卫生组织 (WHO)",
  "year": "2025",
  "language": "zh",
  "disease_scope": ["hypertension", "cardiovascular"],
  "content_type": "public_health",         // guideline | review | public_health | app_doc
  "intended_use": ["lifestyle_advice", "risk_factor_explanation"],
  "clinical_treatment_excluded": true,
  "char_len": 123
}
```

### 6.4 检索（BM25-lite，零依赖、可离线）

- **分词**（中文友好）：ASCII 词 + CJK **一元 + 二元 gram**（无需分词器即可较好召回）。
- **打分**：BM25，`k1=1.5`、`b=0.75`；模块加载时一次性建索引（238 块冷启动可忽略）。
- **流程**：查询 = 用户问题 + 疾病 + Top5 因子标签 → 按 `disease_scope` 过滤（不足则全库兜底）→ 取 **Top-5** → 注入系统 Prompt 的「参考资料」位 → 流式生成后追加 `sources` 事件（标题 + 年份 + 链接）。

### 6.5 安全产物

- **consumer_safe/**：删除药物/治疗/处方后的居民安全版 Markdown。
- **excluded/excluded_segments.md**：被剔除的临床/药物/中医药食方片段清单（审计用）。
- 最终核验：238 块中**零药物/剂量/手术/处方残留**，**每块都带 `source_url` + `year`**。

## 8. 数据流与隐私

```
【问诊】用户回答 → parseAnswer（确定性快路 → 失败走 /api/extract）→ validateInput(内核)
        → store.inputs → 调度下一题 / 本层完成 → 本地 calcCoxRisk → 定性解读(无数字)
【报告】本地 calcCoxRisk + checkDiagnostics + getExclusionReason → 渲染
【为什么/问答】只传 风险等级 + Top5 因子标签 → /api/answer → BM25 检索 → DeepSeek 流式 → sources
```

- **风险计算永远在浏览器本地**；不收集姓名/身份信息。
- 后端密钥（`DEEPSEEK_API_KEY`）只存服务端环境变量，仓库/前端打包产物零泄漏。

## 9. 部署架构

| 入口 | 角色 | 国内访问 | AI 能力 |
|---|---|---|---|
| **GitHub Pages** | 前端（静态） | ✅ | 经 `VITE_API_BASE` 指向 Render 后端 → **完整 AI** |
| **Render**（`server/index.mjs`） | 后端（Node Web Service） | 可达（海外，免费版会休眠） | DeepSeek + RAG |
| **Vercel** | 前端 + 同源 `api/` 函数 | 需代理 | 完整 AI（备用/演示） |

- 前端接口地址可配：`VITE_API_BASE` 空=同源（Vercel/本地 dev）；设=跨域国内后端（Pages+Render）。
- 后端为**零依赖独立服务**（`node:http` + CORS + 读 `PORT`），可同时部署到 Render / 腾讯云 SCF Web 函数 / 阿里云 FC。
